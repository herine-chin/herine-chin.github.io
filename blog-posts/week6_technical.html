Blocs, Procs, and Lambdas

August 2, 2014

Blocs

In ruby, a block is a bit of code that can be executed. The syntax for a block is do and end OR can be contained in those squiggly brackets, { }. A block can be passed into methods like #each.

A proc is a saved block so that we can reuse it easily.

A lambda is just like a proc but with two key differences. Lambdas care about the number of arguments received. Where the proc would ignore and skip faulty arguments, lambdas will throw an error. There's also another key difference between procs and lambdas. When included in a method the proc will execute and then NOT return into the method, while the lambda would return into the method.
